Account Deletion Draft

This page prepares the account deletion request path Ace Domain needs before public mobile store submission.

Pre-release status: destructive account deletion is not active yet. This page is a public draft for the future support/deletion process.

Request Path

Planned request email: support@acedomain.app. The inbox must be created and monitored before this process is used publicly.

A deletion request should include the account email and a clear request to delete the Ace Domain account. Users should never send passwords, recovery codes, API keys, or private tokens.

Identity Verification

Ace Domain should verify account ownership without asking for passwords. Verification can use account email confirmation or another secure support workflow once finalized.

Data Review Needed

  • Supabase auth user identity and profile records.
  • Remote user settings and local device settings guidance.
  • Posts, comments, community memberships, chat messages, and notifications if live.
  • Ace AI request logs only if stored by the hosted backend later.
  • Safety, fraud, moderation, or legal records that may require retention or anonymization.

Important Limits

Freezing, hiding, or logging out of an account is not the same as account deletion. The final workflow must clearly explain what is deleted, retained, anonymized, and how long the request may take.
