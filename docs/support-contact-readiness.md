# Support Contact Readiness

This document prepares Ace Domain for future Play Store/App Store support requirements. It does not create a live support system and does not publish a public support channel yet.

## Recommended Support Email Placeholder

- Placeholder: `support@acedomain.app`
- Status: not confirmed as active.
- Before store submission, replace this placeholder with a real monitored support inbox.

## What Support Contact Must Cover

Ace Domain needs a public support contact that can handle:

- App access and login problems.
- Profile and settings issues.
- Chat, community, and feed issues.
- Safety/reporting concerns.
- Data/privacy questions.
- Account deletion requests.
- Store review contact needs.

## User Issue Categories

### Login / Account Access

- Signup rate-limit questions.
- Email/password login issues.
- Session restore after refresh.
- Demo/local mode confusion.
- Profile sync warnings after login.

### Settings / Profile Issues

- Settings not persisting.
- Profile edits not syncing.
- Local-only settings behavior.
- Theme, chat wallpaper, accessibility, and personalization questions.

### Chats / Communities Issues

- Demo chat fallback questions.
- Live Supabase chat connection issues.
- Community join/leave sync behavior.
- Missing or empty community data.

### AI Assistant Issues

- Ace AI unavailable or quota-limited messages.
- Backend-only OpenAI configuration questions.
- Difference between Ace AI and user-to-user chat.

### Safety / Reporting Issues

- Harassment, scams, spam, or abuse reports.
- Blocking/reporting feature status.
- Community moderation questions.
- Urgent safety escalation instructions before live moderation is complete.

### Data / Privacy / Account Deletion Requests

- Data access requests.
- Profile/account correction requests.
- Account deletion requests.
- Chat/message deletion policy questions.
- Settings/local data questions.

## Ready Now

- Public support page exists at `/support/`.
- Settings Center links to the support page.
- Global Safety includes local report/block controls and support escalation.
- Store prep docs identify the need for support contact and account deletion instructions.
- The app has safe demo/local fallback messaging.
- Auth error messaging is user-friendly and avoids exposing provider internals.
- Safety reminders exist in app surfaces.

## Missing

- Verified monitored support email.
- Hosted public support/help page URL.
- Final privacy policy approval.
- Final account deletion instructions approval.
- Backend account deletion completion workflow.
- Live moderation/report handling workflow beyond local report history.
- Support response process and ownership.

## Before Store Submission

- Create and verify a monitored support inbox.
- Publish a privacy policy/support page.
- Publish account deletion request instructions.
- Add final support, privacy, terms, and account deletion links to the app or public website.
- Prepare test account support instructions for store review.
- Confirm support team/process for safety reports and data requests.

## Current Recommendation

Before submitting to stores, verify the support inbox works and that account deletion requests are processed by a real human/backend workflow. Do not submit with an unmonitored inbox.
